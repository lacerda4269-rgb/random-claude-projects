# 6. Blank Universal Templates — README

**Owner:** Sage (CEO)
**Last Updated:** 2026-06-20

---

## What This Folder Is

This is the single source of truth for every blank template in this version.

Every template that gets deployed during Phase 0 of a new project starts here as a blank copy. When Phase 0 runs, Sophia (or the owning agent) pulls the blank from here, deploys it per the `Getting-Started.md` instructions, and places it in its operational home (Sage's CEO Project Soul, Sophia's COO Folder, etc.). Some blanks get filled out at Phase 0; others deploy empty and are populated as the project runs. Both kinds stage here.

---

## The Rule

> **Blanks live here only.** No blank template is stored in a subfolder alongside the operational file. Subfolders contain only filled/operational files. If you need the blank — come here.

## Scope

> **Blank staging for ALL Phase-0-deployed templates — initialization AND operational.** This folder holds every template that is deployed at Phase 0 of a new project, whether it gets filled in at setup (initialization templates — e.g. the project soul, file index) or deploys empty and is populated as the project runs (operational templates — e.g. trackers, logs, pending-change lists). Both kinds stage here as blanks and travel with every new project. The only files that do *not* belong here are operational workflow forms tied to a specific in-project process (example: the ML Research Request form, which lives in the Master Library with the process it supports) — those belong to their domain folder, not the Phase-0 blank stage.

---

## How to Use

1. Open `Getting-Started.md` (at the version root) — it lists every template, where the blank is, and where the filled file goes.
2. Pull the blank you need from this folder.
3. Fill it out per the README instructions.
4. Place the filled file in its operational destination.
5. The blank stays here unchanged — do not modify originals.
