# 3. Phase 0 & 1

**Owner:** Sage (CEO) with Sophia (COO) executing
**Purpose:** Phase kickoff materials — project setup documents, Phase 0 deliverables, and Phase 1 initialization records that belong to the early project arc.

---

## What This Folder Is

This folder holds the documents produced and consumed during the two phases that stand a project up: Phase 0 (setup and orientation) and Phase 1 (initial build / first deliverable cycle). These documents have a different character from standing ops — they are time-bounded, project-setup-specific, and don't belong in the CEO or COO folders' ongoing operational layers.

**Blank version:** Empty at baseline beyond this README. Phase 0 populates this folder when the project starts.

---

## What Goes Here

- Phase 0 kickoff documents (orientation walk outputs, Phase 0 decisions)
- Phase 0 checklists and setup records
- Phase 1 initialization materials
- Any deliverable explicitly produced *during* Phase 0 or Phase 1 that doesn't have a designated home elsewhere

## What Does NOT Go Here

- Standing operational files (those live in `2. Sophia's COO Folder/1. Operational Files/`)
- Ongoing session artifacts (those live in `1. Sage's CEO Folder/`)
- Research or library materials (those go through the ML pipeline to `5. Master Library/`)
- Work products from Phase 2 onward

---

## Governance

| Rule | Detail |
|------|--------|
| **Activated by** | Sage at project start — folder is always present; Phase 0 work lands here as it is produced |
| **Update authority** | Sage and Sophia. Sophia executes file placement; Sage approves what goes here vs. elsewhere |
| **Structure** | Project-defined — no prescribed subfolder pattern at baseline |

---

*Scope is Phase 0 and Phase 1 only. Once the project moves to Phase 2, this folder is read-only history.*
