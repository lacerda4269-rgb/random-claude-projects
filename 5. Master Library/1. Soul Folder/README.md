## What This Folder Is

The Soul Folder is the agent bench — and as of the residency-model change, it is the home base for every agent except Sophia.

**Residency model:** Only Sophia (COO) ships live on every project — she rides in `2. Sophia's COO Folder/` and activates every session. Every other agent is **on-demand**: they live here in the Soul Folder and are pulled into a project when the work actually calls for them. The old standing-vs-specialty split (separate "Standing Agents" and "Special Project Agents" folders) is retired — there is one flat roster here now, and activation is per-project need, not folder residency.

**This is what "soul carry-forward" means:** Agents grow from project to project through their soul files. Each new project pulls from the highest tier the agent has reached — so every project benefits from every prior project's growth.

> **By design — not residue.** The populated souls in this folder are INTENTIONAL carry-forward. A "blank universal package" sterilizes project *content*, not the team — the roster and its grown souls travel with the package on purpose. Do not flag populated souls as leftover project data in future sterilization reviews; they are the bench, ready for the next project.

> **Feynman — documented intentional exception.** Feynman the Scholar is bound to Jay's real degree (UMass-Dartmouth) for the duration of that degree. That binding is deliberate — he is a live, continuous agent on Jay's actual coursework, not a generic template specialist. The degree-specific references in his soul/SOPs are kept on purpose; do not genericize them.
