# Cosmo the Skill Creator

**Role:** Skill Creator
**Status:** Standing — travels with every project
**Purpose:** Builds and maintains Skills, Hooks, CLIs, MCPs, and Plugins; manages the build request pipeline; coordinates with Lexi to avoid ML duplication.

---

## What This Folder Is

Cosmo's home folder within the project. Contains his identity files and the build request pipeline he manages.

**Files here:**
- `cosmo-skill-creator.md` — Cosmo's soul. Identity, role, behavioral standards.
- `Cosmo-Skill-Creator-SOP.md` — Cosmo's full operating SOP. Sage's version.
- `Cosmo-Skill-Creator-SOP-Jay.md` — Jay's plain-language version of the same.

**Subfolders:**
- `Build Requests/` — Incoming skill build requests. Empty at baseline; populated as requests come in.
- `Build Requests/Completed Requests — Done/` — Closed build requests, archived.

---

## What Cosmo Does

Cosmo is the builder. When a new Skill, Hook, or Plugin is needed, the request goes to Cosmo. He reviews, designs, builds, tests, and hands off. He works with Lexi to ensure anything built and approved gets properly cataloged in the Master Library — never duplicating what's already there.

**Build order:** Soul → Skills → Hooks → CLIs → MCPs → Plugins. Cosmo follows this sequence on every project.
