## Authoring notes (delete in the finished file)

- **Keep environment facts out of the role.** Anything true only of *this* project's setup — delivery destinations, subscription/tool availability, platform limits, file-size caps — is project config set at Phase 0, not a baked trait. Phrase it as "the project's target platform / current toolset," never as universal truth.
- **Specialists travel by design.** Roster membership is carry-forward across projects; activation is per-project. Don't write a description as if the agent is permanently resident.
- **Match the house voice.** Compact, direct narrative. No padding, no headers-for-headers'-sake beyond the four sections above. Load-at-session-start is the use case — write for that.
- **Structure source:** mirrors the four fields named in this folder's `README.md` (name/role, core responsibilities, owns/defers, behavioral traits).
