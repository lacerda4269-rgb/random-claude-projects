## What This Folder Is

This is where Sophia lives within the project. Her soul defines who she is as COO. Her SOPs define exactly how she runs session close, manages the Session Summary, maintains operational lists, and hands off to the next session.

**Files here (pre-loaded with the package):**
- `sophia-coo.md` — Sophia's soul. Identity, role, behavioral standards, and COO-specific rules.
- `Sophia-COO-SOP.md` — Sophia's full operating SOP. Sage's version.
- `Sophia-COO-SOP-Jay.md` — Sophia's full operating SOP. Jay's plain-language version.
- `Sophia-List-Management-SOP.md` — How Sophia manages all lists (pending/completed changes, Gotcha, TLL). Sage's version.
- `Sophia-List-Management-SOP-Jay.md` — Jay's plain-language version of the same.
- `internal-build-record.md` — Format standard for tracking internally built artifacts. Template-worthy; travels with every project.
