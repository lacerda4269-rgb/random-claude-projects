# 1. Session Summary

**Owner:** Sophia (COO) executes; Sage (CEO) owns
**Purpose:** The living record of every session — what happened, what changed, what comes next.

---

## What This Folder Is

The Session Summary is the project's memory across sessions. It captures what was done, decisions made, and the current state at every close. The Index is the navigational companion — one row per session, quick-reference for any past moment.

When the Summary grows too large, it splits into parts. The Index splits at the same boundary — always together, always the same part number.

**Files here (after Phase 0 setup):**
- `Session Summary.md` — Active session log. Current part. Sophia writes; Sage reviews before push.
- `Session Summary Index.md` — Master index across all parts. One row per session.
- Archive folders added as summary parts split.
