# 6. SOPs

**Owner:** Sage (CEO); Sophia (COO) builds and maintains
**Purpose:** All Standard Operating Procedures governing how this project runs — session management, agent creation, close-out, and more.

## What This Folder Is

SOPs define the exact steps for repeatable operations. They are the difference between a process that works once and a process that works every time. Every SOP here has been built and ratified for this project.

**Current SOPs (pre-loaded with the package)** — this table is the SOP registry for the project; every built SOP is listed here.

| File | What It Governs |
|------|----------------|
| `Agent-Creation-Pipeline-SOP.md` | How to build a new agent from soul through Full SOP |
| `CloseOut-HouseKeeping-SOP.md` | End-of-project close-out and housekeeping sequence |
| `Compact-Wrap-Up-SOP.md` | Context compaction wrap-up — what to preserve, what to hand off |
| `CrossTalk-FireDrill-SOP.md` | Cross-agent communication test and fire drill procedure |
| `Every-SOP-Built.md` | Document build methodology — how any SOP/doc is built (Sage's version) |
| `Every-SOP-Built-Jay.md` | Document build methodology — plain-language (Jay's version) |
| `Post-Compact-Return-SOP.md` | Post-compaction return/resume — CP-section read, just-in-time discipline, agent re-activation |
| `Promotion-Pipeline-SOP.md` | How resources move from research → security → library |
| `Session-Close-SOP.md` | Standard session close sequence — summary, push, confirm |
| `Session-Start-SOP.md` | Standard session start sequence — query-first read-in, graphify, activity recall, Sophia activation |

New SOPs built during a project are added to this table — it is the registry. (`Every-SOP-Built.md` is the build methodology, not the registry.)
